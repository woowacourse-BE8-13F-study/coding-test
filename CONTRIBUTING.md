# 참여 방법 (CONTRIBUTING)

## 폴더 구조
```
solutions/<파트번호-파트명>/<문제명>/<깃허브아이디>.java
```
예)
```
solutions/05-배열/배열정렬하기/Uechann.java
```
- 파트번호-파트명은 `plan/` 문서명과 동일하게 맞춥니다(예: `05-배열`, `10-집합-유니온파인드`).
- 문제명 폴더는 공백을 빼고 적습니다(예: `배열정렬하기`).

## 커밋 메시지
```
solve: 05-배열 배열정렬하기 (Uechann)
```

## 브랜치 / PR 규칙 (기본안 — 스터디 합의로 조정)
- **기본안 A (권장):** 개인 브랜치 → PR → 리뷰 후 `main` 병합.
  - 브랜치명: `solve/<깃허브아이디>/<파트번호-문제명>` (예: `solve/Uechann/05-배열정렬하기`)
- **기본안 B:** 소규모 스터디라면 `main`에 직접 push도 허용. (충돌 주의)
- 둘 중 하나를 스터디 첫 주에 정해 이 문서에 확정해 두세요.

## 풀이 코드 규칙
- 각 풀이 **상단 주석**에 다음을 남깁니다:
  - 접근 방법 1~2줄
  - 시간복잡도(가능하면 공간복잡도도)
```java
// 접근: 정렬 후 투 포인터로 합이 target인 쌍 탐색
// 시간복잡도: O(n log n)
public class Solution { /* ... */ }
```
- 통과 후, 해당 문제의 체크박스([ ] → [x])를 `plan/`과 `schedule/`, `README`에서 체크합니다.
