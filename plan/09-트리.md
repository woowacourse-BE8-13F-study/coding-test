# 09. 트리

## 📚 개념 정리 (문제 풀기 전 필독)

> 교재 목차 소제목: 트리 개념 · 이진 트리 표현(배열·포인터·인접 리스트) · 이진 트리 순회 · 이진 탐색 트리 구축·탐색·효율 비교

- 트리는 사이클이 없는 계층형 자료구조로, 노드와 간선으로 부모-자식 관계를 표현한다.
- 이진 트리는 각 노드가 최대 두 자식을 가지며 순회 방식(전위/중위/후위)에 따라 방문 순서가 달라진다.
- 이진 탐색 트리(BST)는 왼쪽<루트<오른쪽 규칙으로 평균 O(log n) 탐색을 지원한다.

### 꼭 알아야 할 선행 지식
- 재귀
- 이진 트리 표현(배열/포인터/인접 리스트)
- 순회(DFS 계열)

### 핵심 연산의 시간복잡도

| 연산 | 시간복잡도 |
|---|---|
| BST 탐색(균형) | O(log n) |
| BST 탐색(편향) | O(n) |
| 순회(전체) | O(n) |
| 삽입(균형 BST) | O(log n) |

### 자바 구현 포인트
- 배열 표현: 노드 i의 자식은 `2i+1`, `2i+2` (완전 이진 트리에 적합).
- 포인터 표현: `class Node { int val; Node left, right; }`.
- 순회는 보통 재귀; 깊이가 깊으면 명시적 스택으로 스택오버플로우 회피.

### 자주 하는 실수 / 주의점
- 전위/중위/후위 순서 혼동
- 편향 트리에서 O(n)이 되는 점 간과
- 재귀 깊이 초과
