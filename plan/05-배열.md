# 05. 배열

## 📚 개념 정리 (문제 풀기 전 필독)

> 교재 목차 소제목: 배열 선언 · 배열과 차원 · 자바 배열 사용법(데이터 추가) · 배열 연산 시간복잡도 · 배열 선택 시 고려점

- 배열은 같은 타입의 데이터를 메모리에 연속으로 저장하는 자료구조로, 인덱스로 O(1) 임의 접근이 가능하다.
- 크기가 고정되어 있고 중간 삽입/삭제 시 원소 이동 비용이 발생한다.
- 2차원 이상 배열은 행렬·격자 문제에서 좌표 접근의 기본이 된다.

### 꼭 알아야 할 선행 지식
- 인덱스와 0-based 접근
- 다차원 배열 선언과 순회
- 얕은 복사 vs 깊은 복사

### 핵심 연산의 시간복잡도

| 연산 | 시간복잡도 |
|---|---|
| 임의 접근(인덱스) | O(1) |
| 맨 뒤 추가(여유 有) | O(1) |
| 중간 삽입/삭제 | O(n) |
| 탐색(정렬 안 됨) | O(n) |

### 자바 구현 포인트
- 고정 크기: `int[] a = new int[n];`, 가변 길이는 `ArrayList` 또는 `int[]` 재할당.
- 2차원: `int[][] m = new int[r][c];` (자바는 배열의 배열).
- 정렬은 `Arrays.sort(a)`, 복사는 `Arrays.copyOf` / `System.arraycopy`.
- `Arrays.toString` / `Arrays.deepToString` 으로 디버깅.

### 자주 하는 실수 / 주의점
- 인덱스 범위 초과(IndexOutOfBounds)
- 2차원 배열의 행/열 순서 혼동
- 배열 복사 없이 참조만 넘겨 원본이 바뀌는 실수
