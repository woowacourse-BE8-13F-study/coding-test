# 06. 스택

## 📚 개념 정리 (문제 풀기 전 필독)

> 교재 목차 소제목: 스택 동작 원리 · 스택 ADT · Stack/Deque 클래스 사용

- 스택은 후입선출(LIFO) 자료구조로, 가장 최근에 넣은 데이터가 가장 먼저 나온다.
- 괄호 검사, 수식 계산, 뒤로 가기, DFS 등 '되돌리기/짝 맞추기' 문제에 강하다.
- 핵심 연산 push/pop/peek 은 모두 상수 시간이다.

### 꼭 알아야 할 선행 지식
- LIFO 개념
- 스택 오버/언더플로우
- 재귀와 스택의 관계

### 핵심 연산의 시간복잡도

| 연산 | 시간복잡도 |
|---|---|
| push | O(1) |
| pop | O(1) |
| peek | O(1) |
| 탐색 | O(n) |

### 자바 구현 포인트
- `Deque<Integer> stack = new ArrayDeque<>();` 권장 (레거시 `Stack`보다 빠름).
- push=`push`/`addLast`, pop=`pop`/`pollLast`, peek=`peek`/`peekLast`.
- `Stack` 클래스는 `Vector` 기반이라 동기화 오버헤드가 있어 지양.

### 자주 하는 실수 / 주의점
- 빈 스택에서 pop/peek 호출
- `ArrayDeque`에 null 삽입 불가
- peek과 pop 혼동
