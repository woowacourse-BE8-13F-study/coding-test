# 08. 해시

## 📚 개념 정리 (문제 풀기 전 필독)

> 교재 목차 소제목: 해시 개념 · 해시 함수(고려사항·자주 쓰는 함수) · 충돌 처리(체이닝·개방주소법) · 해시맵 · HashMap 클래스

- 해시는 키를 해시 함수로 인덱스에 매핑해 평균 O(1) 삽입/탐색을 제공한다.
- '무언가를 세거나 존재 여부를 빠르게 확인'하는 문제의 1순위 도구다.
- 서로 다른 키가 같은 버킷에 오는 충돌은 체이닝·개방주소법으로 처리한다.

### 꼭 알아야 할 선행 지식
- 해시 함수와 충돌
- 체이닝 vs 개방주소법
- hashCode/equals 규약

### 핵심 연산의 시간복잡도

| 연산 | 시간복잡도 |
|---|---|
| 삽입(평균) | O(1) |
| 탐색(평균) | O(1) |
| 삭제(평균) | O(1) |
| 최악(충돌 집중) | O(n) |

### 자바 구현 포인트
- 카운팅/매핑: `HashMap<K,V>`, 존재 확인: `HashSet<K>`.
- `getOrDefault(k,0)+1` 로 빈도 세기, `merge(k,1,Integer::sum)` 도 유용.
- 정렬된 순회가 필요하면 `TreeMap`, 삽입 순서 유지가 필요하면 `LinkedHashMap`.

### 자주 하는 실수 / 주의점
- 사용자 정의 키에서 hashCode/equals 미구현
- 값 타입 int/Integer 오토박싱 비교(==) 실수
- 정렬이 필요한데 HashMap 순회 결과가 무순서인 점 간과
